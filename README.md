# Alexa Device Builder

A Home Assistant custom integration (installable via [HACS](https://hacs.xyz/)) that lets you configure which Home Assistant devices are exposed to Alexa — directly from the Home Assistant UI — without needing the Home Assistant Cloud subscription.

It writes the Alexa smart home filter configuration to a YAML package file automatically whenever you save your selection.

---

## Use case

This integration is designed for users who:
- Self-host Home Assistant and expose it securely via a public domain
- Use an **AWS Lambda function** to forward Alexa Smart Home skill requests to their Home Assistant instance
- Do **not** use Nabu Casa / Home Assistant Cloud

The standard Alexa integration in Home Assistant requires filter configuration in `configuration.yaml` (or a package file). This integration provides a friendly GUI to manage that configuration.

---

## Prerequisites

### 1. Home Assistant Alexa Smart Home skill

Follow the [Home Assistant Alexa Smart Home documentation](https://www.home-assistant.io/integrations/alexa.smart_home/) to set up the Alexa Developer Console skill and AWS Lambda function pointing to your Home Assistant instance.

### 2. Enable packages in `configuration.yaml`

The integration writes its output to a YAML package file. Make sure your `configuration.yaml` includes the packages directory:

```yaml
homeassistant:
  packages: !include_dir_named packages
```

If the `packages/` directory does not exist in your config directory, the integration will create it automatically.

## Installation

### Via HACS (recommended)

1. Open HACS in Home Assistant
2. Go to **Integrations**
3. Click **⋮ → Custom repositories**
4. Add `https://github.com/nikodemusz/HACS-AlexaDeviceBuilder` as an **Integration**
5. Search for **Alexa Device Builder** and install it
6. Restart Home Assistant

### Manual installation

1. Copy the `custom_components/alexa_device_builder/` folder into your Home Assistant `custom_components/` directory
2. Restart Home Assistant

## Configuration

### Step 1 – Add the integration

1. Go to **Settings → Devices & Services → Add Integration**
2. Search for **Alexa Device Builder**
3. Choose the **Operation mode**:
   - **Home Assistant YAML package**
   - **Amazon account management** (separate device-management sync)
4. If you selected YAML mode, set the package file path (default: `packages/alexa_devices.yaml`)
5. Complete setup

### Step 2A – Configure Home Assistant device exposure (YAML mode)

1. On the integration card, click **Configure**
2. Select entities and/or domains using the four filter lists:
   - **Include entities** – expose only these specific entities
   - **Include domains** – expose all entities in these domains
   - **Exclude entities** – hide these specific entities from Alexa
   - **Exclude domains** – hide all entities in these domains from Alexa
3. Click **Submit** — the package file is written immediately

> **Tip:** Leave all lists empty to expose every entity to Alexa.

> **Note:** After changing the filter, you need to **restart Home Assistant** for the new configuration to take effect. The `alexa` integration is configured via YAML and does not support hot-reload at runtime.

### Step 2B – Configure Amazon-side device management (Amazon mode)

In Amazon mode, device management is configured independently from HA entity exposure:

- `amazon_region` selects the Amazon marketplace.
- `amazon_devices` is a YAML mapping for devices that are already available in Alexa and should be edited or marked for removal.
- Runtime sync requires an active **Alexa Media Player** account session in Home Assistant.

Example:

```yaml
device_id_1: New Name
device_id_2:
  remove: true
device_id_3:
  name: Kitchen Light
  remove: false
```

Removing a mapping entry stops managing that device in this integration.  
Setting `remove: true` will try to remove that Alexa device during sync.

Sync runs when:
- the integration starts
- Amazon mode options are saved

## Generated package file

The integration writes a file like this:

```yaml
# Generated by Alexa Device Builder - do not edit manually
alexa:
  smart_home:
    filter:
      include_entities:
        - light.living_room
        - switch.bedroom_lamp
      include_domains:
        - light
      exclude_entities:
        - sensor.internal_temperature
      exclude_domains:
        - binary_sensor
```

---

## Filter rules

HA Alexa filter rules follow this priority:

1. If any `include_*` list is set, **only** those entities/domains are exposed.
2. `exclude_*` lists further remove entities from the exposed set.
3. If all lists are empty, **all** entities are exposed.

See the [Home Assistant Alexa filter documentation](https://www.home-assistant.io/integrations/alexa.smart_home/#filter-configuration) for full details.
